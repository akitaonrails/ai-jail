# v1.3.0

## WSL Docker Desktop compatibility release

This release fixes Docker CLI discovery inside `ai-jail` on WSL 2 systems that use Docker Desktop for Windows.

### Docker Desktop WSL CLI tools

- Docker passthrough still auto-enables when `/var/run/docker.sock` exists.
- On WSL 2, Docker Desktop commonly installs `/usr/bin/docker` as a symlink to `/mnt/wsl/docker-desktop/cli-tools/usr/bin/docker`.
- ai-jail already mounted `/usr` and the Docker socket, but not that `/mnt/wsl/...` symlink target, so `docker` could appear missing inside the jail.
- When the Docker socket exists and Docker Desktop's WSL CLI tools directory is present, ai-jail now mounts `/mnt/wsl/docker-desktop/cli-tools` read-only.

### Documentation and tests

- Added WSL 2 Docker Desktop troubleshooting notes to the README.
- Added bwrap unit tests for Docker socket + WSL CLI tool discovery.

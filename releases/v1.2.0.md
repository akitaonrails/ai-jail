# v1.2.0

## Oh My Pi compatibility release

This release fixes `ai-jail omp` exiting immediately for OMP installs that write runtime state under `~/.omp`.

### OMP home state is writable

- Added `~/.omp` to the read-write AI tool state directory list.
- `omp` already resolved correctly through `~/.bun/bin`, Bun, Node, and mise; the failure was the sandbox mounting `~/.omp` read-only while OMP tried to open `~/.omp/logs/omp.<date>.log`.
- Kept sensitive home dotdirs denied and preserved the existing default-private-home behavior.

### Regression coverage

- Added a bwrap regression test proving `~/.omp` is emitted as a read-write bind mount.
- Extended dotdir allowlist tests so `~/.omp` cannot regress back to read-only or user-deniable behavior.
- Isolated sandbox escape tests from ambient `.ai-jail` settings so release verification is not affected by a developer project enabling SSH passthrough.

### Verification

- `cargo fmt --check`
- `cargo clippy -- -D warnings`
- `cargo test`
- `cargo run -- --version`
- `cargo run -- --exec omp --version`

# v1.10.0

## Config, path denial, and host integration improvements

### Added

- Added Pi agent preset support, including writable `~/.pi` and `~/.pi-lens` state directories (#65).
- Added global command-scoped config tables in `~/.ai-jail`, such as `[commands.pi]`, while keeping project `.ai-jail` files flat and backward-compatible (#68).
- Added `deny_paths` / `--deny-path` for permission-denied path overlays when empty `--mask` placeholders would mislead agents (#67).
- Added dangerous opt-in `systemd_user = true` / `--systemd-user` support for normal Linux mode, exposing the host user D-Bus/systemd manager sockets for `systemd-run --user` (#71).
- Added `.thunderbird` to the built-in sensitive home dotdir deny list (#72).

### Fixed

- Refuse `/` in `ro_maps` / `rw_maps` so user extra mounts cannot shadow core sandbox mounts such as `/dev` (#70).
- Project auto-save now keeps inherited global config out of project `.ai-jail` files while still using the global values at runtime (#69).
- Cleaned up post-refactor test/process handling so stricter all-target clippy passes without zombie-process warnings.

### Safety notes

- `--systemd-user` is skipped under lockdown and browser profile mode, and remains off by default.
- Existing `.ai-jail` files remain compatible. New config fields default safely and unknown fields remain ignored.

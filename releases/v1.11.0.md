# v1.11.0

## macOS 26 restricted-modes fix + exec-mode config hygiene

Everything that landed since v1.10.3: two user-facing bug fixes from
@zanlucathiago and a CI guard that keeps macOS lint drift out of
master.

### Fixed

- **macOS 26 (Tahoe): `--private-home`, `--lockdown`, and `--browser`
  no longer abort with SIGABRT before the command runs** (#77, #78).
  The restricted read allow-list never granted the root node `/`, which
  the dyld loader on macOS 26 needs to resolve absolute paths. The fix
  is a `literal "/"` rule — deliberately not `subpath "/"`, so only the
  root directory listing is exposed and the restricted allow-list stays
  intact.
- A one-off `ai-jail --exec ./script` no longer persists
  `no_status_bar = true` into the global `$HOME/.ai-jail` (#79, #80).
  `--exec` forces the status bar off for clean stdout, and that forced
  value was indistinguishable from an explicit `--no-status-bar`
  preference. `--exec` is now excluded from the global preference save,
  the same way `--dry-run` already was. Plain `--no-status-bar` still
  persists as intended.
- Cleared four clippy `-D warnings` failures in macOS-only code so the
  CLAUDE.md pre-commit gate passes on a clean checkout on macOS
  (#75, #76).

### Changed

- CI now lints the macOS target with clippy from the Linux runner
  (`cargo clippy --target aarch64-apple-darwin --all-targets`),
  replacing the weaker `cargo check` cross-compile step that let the
  macOS-only warnings drift in unnoticed (#75).

### Added

- Regression test that all three restricted modes (private-home,
  lockdown, browser) grant the root-node read and never `subpath "/"`.
- Regression tests that `--exec` never persists global status-bar
  preferences, for both the forced `status_bar` value and an explicit
  `--status-bar=STYLE`.

### Validation

- Verified formatting, clippy (Linux and aarch64-apple-darwin
  cross-target, all targets), and the full test suite before release.
- Reproduced and re-verified the `--exec` config write with an isolated
  `$HOME` on Linux: no global write in `--exec` and
  `--no-save-config --exec`, preference still saved with a plain
  `--no-status-bar`.
- The macOS 26 SIGABRT fix was live-tested by the author on macOS
  26.5.2 (arm64): restricted modes run, project writes land, reads
  outside the allow-list stay denied.

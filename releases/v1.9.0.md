# v1.9.0

## Tailscale DNS and socket support

### Fixed

- Fixed Tailscale split-DNS lookups inside the sandbox (#62). When the host uses systemd-resolved and the resolver search/domain list contains Tailscale MagicDNS domains such as `ts.net` or `*.ts.net`, ai-jail now keeps the systemd-resolved stub resolver so per-domain routing is preserved.

### Added

- Added opt-in `--tailscale` / `--no-tailscale` and `tailscale = true` config support (#63).
- When enabled, ai-jail maps `/var/run/tailscale/tailscaled.sock` into the sandbox if it exists, allowing the `tailscale` CLI to talk to the host daemon.

### Safety notes

- Tailscale socket passthrough is off by default and disabled under `--lockdown`.
- Existing `.ai-jail` config files remain compatible; the new `tailscale` field is optional and defaults to off.

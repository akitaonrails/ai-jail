# v1.10.3

## Cross-PR post-audit hardening

Maintenance release from a cross-cutting audit of everything that
landed between v1.9.0 and v1.10.2.

### Fixed

- Granted the Tailscale socket read-write in the Landlock ruleset,
  mirroring the Docker socket rule. Landlock ABI v3 does not currently
  mediate unix-socket connect(), so this is layer consistency and
  defense-in-depth rather than a user-visible fix — the policy stays
  correct if a future Landlock ABI starts mediating socket access.
- `--tailscale` now prints a platform note on macOS instead of being
  silently ignored (no socket bind exists there).

### Changed

- The systemd user-bus socket subpaths and the Tailscale socket path
  are now shared constants between the bwrap mount layer and the
  Landlock rule layer, so the two can no longer drift apart.
- Named the `ts.net` MagicDNS TLD constant, documented the hand-rolled
  glob's deliberate limits (no `[!...]` negation), and deduplicated the
  saved-config header string.

### Added

- Regression test that an old plain global `~/.ai-jail` (written
  before `[commands.*]` tables existed) round-trips through a save
  without losing fields.
- Wiring test that the Tailscale socket bind is never emitted when the
  feature is disabled or the socket is absent.

### Validation

- Verified formatting, clippy (including all targets), unit and
  integration tests, macOS target check, and live sandbox smoke tests
  (including `tailscale status` inside the jail under fully enforced
  Landlock) before release.

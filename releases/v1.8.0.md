# v1.8.0

## Copy-on-write overlay maps

### Added

- Added `--overlay-map <PATH>` / `overlay_maps` for copy-on-write mounts (#58).
- Overlay maps let the sandbox see a directory as writable while all changes land under `<project>/.ai-jail-overlays/`, leaving the original directory untouched.
- Overlay storage is created eagerly, including for `--dry-run`, because the upper/work directories must exist before OverlayFS can mount them.

### Safety notes

- Overlay maps are opt-in only.
- Overlay maps are disabled under `--lockdown` and browser isolation profiles.
- On macOS, overlay maps degrade to read-only mounts with a warning.
- The `.ai-jail-overlays/` storage directory is hidden from the sandbox and contains its own `.gitignore`.

# v1.14.0

## Safer configuration and more precise path control

This release fails closed when a sandbox policy cannot be loaded and adds two
ways to express paths more precisely: alternate destinations for maps and
explicit exceptions for broad mask/deny patterns.

### Security

- **Existing project and global `.ai-jail` files now fail closed when they
  cannot be read, decoded as UTF-8, or parsed as TOML** (#87). ai-jail exits
  before merging defaults, rewriting configuration, building a dry run, or
  launching the sandbox. Fatal errors remain visible under `--exec`. Missing
  files still use defaults, and `--clean --init` remains the explicit recovery
  path for replacing a malformed project config. Reported by @mobiuscog.
- Global status-bar preference updates also refuse to overwrite a config that
  becomes malformed before the save-time re-read.

### Added

- **`--map` and `--rw-map` now accept `SOURCE:DESTINATION`** (#85), including
  the `ro_maps` and `rw_maps` config fields. Both sides independently expand
  `~` and resolve relative paths against the project directory. Only mounts
  successfully created by bubblewrap are forwarded to Landlock. Contributed
  by @ohmer1.
- **`--mask-except` and `--deny-path-except` subtract explicit paths or globs
  from expanded mask/deny results** (#86). The corresponding config fields are
  `mask_exceptions` and `deny_path_exceptions`. Literal exceptions include
  descendants; glob exceptions use project-relative semantics. Requested by
  @mdindoffer.

### Changed

- Alternate map destinations require Linux/bubblewrap and fail explicitly on
  macOS, where `sandbox-exec` cannot remap paths. Existing same-path maps remain
  compatible on both platforms.
- Documentation now positions the default mode as the intended mode for AI
  agents. Private-home deliberately starts agents without their host config or
  credentials unless command-scoped mappings grant that state explicitly.
- `--clean` help now states that it ignores only the project config; the global
  config is still validated.

### Validation

- Verified formatting, clippy with warnings denied, the complete Linux test
  suite, and aarch64-apple-darwin all-target clippy before release.
- Added process-level regression tests for malformed project/global configs,
  `--exec` error visibility, non-overwrite guarantees, and explicit
  `--clean --init` recovery.

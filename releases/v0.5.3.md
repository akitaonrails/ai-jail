# v0.5.3

## Fix: status bar glitches with TUI children (Claude Code, vim, etc.)

The PTY rendering pipeline now splits child output at the exact byte where a scroll-region reset occurs, re-establishing the status bar before forwarding the remaining bytes. Previously, the entire output chunk was forwarded first and the status bar re-established afterwards — content after the reset rendered on the unrestricted terminal, causing visual artifacts (stray box-drawing characters, misplaced content) during TUI redraws.

This primarily affected Claude Code, which resets the scroll region as part of its normal UI rendering cycle.

### Technical details

- New `scan_first_reset()` method returns the byte offset of the first reset-triggering sequence (`\x1b[r`, `\x1bc`, alt-screen switches).
- New `forward_child_chunk()` loop: write up to reset → redraw status bar → process remainder. Handles multiple resets per chunk.
- The `ResetDetector` state machine advances incrementally, allowing repeated calls on successive slices of the same buffer.
